## 任务描述

创建标准的Go项目目录结构，包括cmd/、internal/、pkg/、api/、config/、migrations/、scripts/、test/等目录。确保目录结构清晰，符合Go项目最佳实践。

## 依赖任务

无

## 实现指南

### 1. 创建根目录结构

创建以下目录结构：

```
LNCms/
├── cmd/
│   └── core/              # 核心模块入口
├── internal/
│   ├── api/               # API处理器
│   ├── config/            # 配置管理
│   ├── database/          # 数据库连接
│   ├── cache/             # Redis缓存
│   ├── search/            # Elasticsearch搜索
│   ├── middleware/        # 中间件
│   └── errors/            # 错误处理
├── pkg/                   # 公共包
├── api/                   # API路由定义
│   ├── v1/                # API v1版本
│   └── proto/             # gRPC proto文件
├── config/                # 配置文件
├── migrations/            # 数据库迁移
├── scripts/               # 脚本文件
└── test/                  # 测试文件
```

### 2. 创建必要的空文件占位

为每个目录创建 `.gitkeep` 文件，确保空目录也能被Git跟踪：

```bash
# 为每个目录创建 .gitkeep 文件
touch cmd/core/.gitkeep
touch internal/api/.gitkeep
touch internal/config/.gitkeep
touch internal/database/.gitkeep
touch internal/cache/.gitkeep
touch internal/search/.gitkeep
touch internal/middleware/.gitkeep
touch internal/errors/.gitkeep
touch pkg/.gitkeep
touch api/v1/.gitkeep
touch api/proto/.gitkeep
touch config/.gitkeep
touch migrations/.gitkeep
touch scripts/.gitkeep
touch test/.gitkeep
```

### 3. 验证目录结构完整性

检查所有目录是否已正确创建，确保目录结构符合Go项目标准。

## 验收标准

1. **所有必需的目录已创建**
   - cmd/core/ 目录存在
   - internal/ 下所有子目录存在
   - pkg/ 目录存在
   - api/ 目录及其子目录存在
   - config/ 目录存在
   - migrations/ 目录存在
   - scripts/ 目录存在
   - test/ 目录存在

2. **目录结构符合Go项目标准**
   - 遵循Go项目标准目录结构
   - 目录命名使用小写字母和下划线
   - 目录层次清晰，职责明确

3. **目录命名清晰，符合项目规范**
   - 目录名称具有描述性
   - 符合Go项目命名约定
   - 便于后续开发和维护

## 相关文件

- `cmd/core/` - 核心模块入口目录
- `internal/` - 内部代码目录
- `pkg/` - 公共包目录
- `api/` - API路由目录
- `config/` - 配置文件目录
- `migrations/` - 数据库迁移目录
- `scripts/` - 脚本文件目录
- `test/` - 测试文件目录
