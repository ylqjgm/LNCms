## 任务描述

## 实现指南

### 1. 创建.env.example文件

在项目根目录创建 `.env.example` 文件。

### 2. 列出所有可配置的环境变量

按照配置模块分组列出所有可配置的环境变量：

```bash
# ==================== 服务器配置 ====================
# HTTP服务器端口
SERVER_HTTP_PORT=8080

# gRPC服务器端口
SERVER_GRPC_PORT=9090

# 运行模式：dev、test、prod
SERVER_MODE=dev

# ==================== 数据库配置 ====================
# 数据库主机
DATABASE_HOST=localhost

# 数据库端口
DATABASE_PORT=5432

# 数据库用户名
DATABASE_USER=postgres

# 数据库密码
DATABASE_PASSWORD=password

# 数据库名称
DATABASE_NAME=lncms

# 最大打开连接数
DATABASE_MAX_OPEN_CONNS=25

# 最大空闲连接数
DATABASE_MAX_IDLE_CONNS=5

# 连接最大生存时间（秒）
DATABASE_CONN_MAX_LIFETIME=300

# ==================== Redis配置 ====================
# Redis主机
REDIS_HOST=localhost

# Redis端口
REDIS_PORT=6379

# Redis密码（如果设置了密码）
REDIS_PASSWORD=

# Redis数据库编号
REDIS_DB=0

# 连接池大小
REDIS_POOL_SIZE=10

# ==================== Elasticsearch配置 ====================
# Elasticsearch URL列表（逗号分隔）
ELASTICSEARCH_URLS=http://localhost:9200

# Elasticsearch用户名（如果需要认证）
ELASTICSEARCH_USERNAME=

# Elasticsearch密码（如果需要认证）
ELASTICSEARCH_PASSWORD=

# 索引前缀
ELASTICSEARCH_INDEX_PREFIX=lncms

# ==================== 日志配置 ====================
# 日志级别：debug、info、warn、error
LOG_LEVEL=info

# 日志输出方式：stdout、file
LOG_OUTPUT=stdout

# 日志文件路径（当输出方式为file时）
LOG_PATH=./logs/app.log

# 日志格式：json、text
LOG_FORMAT=json
```

### 3. 为每个环境变量添加注释说明

为每个环境变量添加清晰的中文注释，说明：
- 环境变量的用途
- 取值范围或格式要求
- 是否为必填项
- 默认值

### 4. 设置合理的默认值

为每个环境变量设置合理的默认值：
- 不包含敏感信息（如密码、密钥等）
- 使用占位符或示例值
- 默认值适合开发环境

### 5. 按照配置模块分组

按照配置模块分组组织环境变量：
- 服务器配置
- 数据库配置
- Redis配置
- Elasticsearch配置
- 日志配置

### 6. 使用标准的.env文件格式

使用标准的.env文件格式：
- 使用 `KEY=VALUE` 格式
- 支持注释（以 `#` 开头）
- 支持空行分隔

### 7. 确保.env文件在.gitignore中

确保 `.env` 文件在 `.gitignore` 中，避免提交敏感信息：
- `.env` 文件包含真实配置和敏感信息
- `.env.example` 文件作为模板，不包含敏感信息
