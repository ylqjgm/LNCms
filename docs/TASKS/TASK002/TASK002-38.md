## 任务描述

设计核心模块的作者收益表（author_earnings），用于存储作者收益记录信息。必须遵循 PostgreSQL 数据库设计规范，包含完整的字段定义、约束、索引、注释等。

## 依赖任务

TASK002-13（作者表）、TASK002-37（流水表）

## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 和 `docs/Features/Core.md` 文档，了解作者收益管理的详细需求：
- 作者收益管理：收益查询、收益统计、收益审核、收益支付
- 作者收益关联作者表和流水表，记录收益来源
- 作者收益需要支持多种收益类型：全勤收益、订阅收益、阅读收益、收藏收益、票务收益、打赏收益、新手鼓励
- 作者收益主要使用虚拟货币（消费货币）进行计算和存储，需要支持虚拟货币收益金额、虚拟货币佣金比例、虚拟货币佣金金额、虚拟货币净收益金额等
- 作者收益需要支持现实金额字段（可选），用于结算时记录转换后的现实金额，方便统计分析和结算管理
- 作者收益需要支持收益状态、结算时间、结算周期等

### 2. 设计作者收益表结构

设计 `author_earnings` 表，包含作者收益基本信息：

```sql
CREATE TABLE author_earnings (
    id BIGSERIAL PRIMARY KEY,
    author_id BIGINT NOT NULL,
    transaction_id BIGINT NOT NULL,
    earning_type VARCHAR(20) NOT NULL,
    virtual_currency_earning_amount NUMERIC(10, 2) NOT NULL,
    virtual_currency_commission_rate NUMERIC(5, 2) NOT NULL DEFAULT 0.00,
    virtual_currency_commission_amount NUMERIC(10, 2) NOT NULL DEFAULT 0.00,
    virtual_currency_net_amount NUMERIC(10, 2) NOT NULL,
    real_amount NUMERIC(10, 2),
    real_commission_amount NUMERIC(10, 2),
    real_net_amount NUMERIC(10, 2),
    earning_status VARCHAR(20) NOT NULL DEFAULT 'pending',
    settled_at TIMESTAMPTZ,
    settlement_period VARCHAR(20),
    remark TEXT,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_author_earnings_author_id FOREIGN KEY (author_id) REFERENCES authors(id) ON DELETE CASCADE,
    CONSTRAINT fk_author_earnings_transaction_id FOREIGN KEY (transaction_id) REFERENCES transactions(id) ON DELETE RESTRICT,
    CONSTRAINT ck_author_earnings_earning_type CHECK (earning_type IN ('attendance', 'subscription', 'reading', 'collection', 'ticket', 'reward', 'newbie_bonus')),
    CONSTRAINT ck_author_earnings_virtual_currency_earning_amount CHECK (virtual_currency_earning_amount > 0),
    CONSTRAINT ck_author_earnings_virtual_currency_commission_rate CHECK (virtual_currency_commission_rate >= 0 AND virtual_currency_commission_rate <= 100),
    CONSTRAINT ck_author_earnings_virtual_currency_commission_amount CHECK (virtual_currency_commission_amount >= 0),
    CONSTRAINT ck_author_earnings_virtual_currency_net_amount CHECK (virtual_currency_net_amount > 0),
    CONSTRAINT ck_author_earnings_real_amount CHECK (real_amount IS NULL OR real_amount > 0),
    CONSTRAINT ck_author_earnings_real_commission_amount CHECK (real_commission_amount IS NULL OR real_commission_amount >= 0),
    CONSTRAINT ck_author_earnings_real_net_amount CHECK (real_net_amount IS NULL OR real_net_amount > 0),
    CONSTRAINT ck_author_earnings_earning_status CHECK (earning_status IN ('pending', 'settled', 'cancelled'))
);

-- 创建索引
CREATE INDEX idx_author_earnings_author_id ON author_earnings(author_id);
CREATE INDEX idx_author_earnings_transaction_id ON author_earnings(transaction_id);
CREATE INDEX idx_author_earnings_earning_type ON author_earnings(earning_type);
CREATE INDEX idx_author_earnings_earning_status ON author_earnings(earning_status);

CREATE INDEX idx_author_earnings_settlement_period ON author_earnings(settlement_period);
CREATE INDEX idx_author_earnings_created_at ON author_earnings(created_at);
CREATE INDEX idx_author_earnings_virtual_currency_earning_amount ON author_earnings(virtual_currency_earning_amount);
CREATE INDEX idx_author_earnings_virtual_currency_net_amount ON author_earnings(virtual_currency_net_amount);
CREATE INDEX idx_author_earnings_real_amount ON author_earnings(real_amount) WHERE real_amount IS NOT NULL;
CREATE INDEX idx_author_earnings_real_net_amount ON author_earnings(real_net_amount) WHERE real_net_amount IS NOT NULL;
CREATE INDEX idx_author_earnings_author_status ON author_earnings(author_id, earning_status);
CREATE INDEX idx_author_earnings_author_settled ON author_earnings(author_id, settled_at);
CREATE INDEX idx_author_earnings_author_period ON author_earnings(author_id, settlement_period);
CREATE INDEX idx_author_earnings_author_type_amount ON author_earnings(author_id, earning_type, virtual_currency_net_amount);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_author_earnings_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_author_earnings_updated_at
    BEFORE UPDATE ON author_earnings
    FOR EACH ROW
    EXECUTE FUNCTION update_author_earnings_updated_at();
```

### 3. 设计约束

- **主键约束**：`pk_author_earnings` (id)
- **外键约束**：`fk_author_earnings_author_id` (author_id) 关联 authors(id)，级联删除
- **外键约束**：`fk_author_earnings_transaction_id` (transaction_id) 关联 transactions(id)，删除时禁止（RESTRICT）
- **检查约束**：`ck_author_earnings_earning_type` (earning_type) - 确保收益类型是预定义的值
- **检查约束**：`ck_author_earnings_virtual_currency_earning_amount` (virtual_currency_earning_amount) - 确保虚拟货币收益金额大于0
- **检查约束**：`ck_author_earnings_virtual_currency_commission_rate` (virtual_currency_commission_rate) - 确保虚拟货币佣金比例在0-100之间
- **检查约束**：`ck_author_earnings_virtual_currency_commission_amount` (virtual_currency_commission_amount) - 确保虚拟货币佣金金额大于等于0
- **检查约束**：`ck_author_earnings_virtual_currency_net_amount` (virtual_currency_net_amount) - 确保虚拟货币净收益金额大于0
- **检查约束**：`ck_author_earnings_real_amount` (real_amount) - 确保现实金额大于0（如果存在）
- **检查约束**：`ck_author_earnings_real_commission_amount` (real_commission_amount) - 确保现实佣金金额大于等于0（如果存在）
- **检查约束**：`ck_author_earnings_real_net_amount` (real_net_amount) - 确保现实净收益金额大于0（如果存在）
- **检查约束**：`ck_author_earnings_earning_status` (earning_status) - 确保收益状态是预定义的值
- **非空约束**：author_id、transaction_id、earning_type、virtual_currency_earning_amount、virtual_currency_net_amount、earning_status 必须非空
- **默认值**：virtual_currency_commission_rate 默认为 0.00，virtual_currency_commission_amount 默认为 0.00，earning_status 默认为 'pending'
- **可选字段**：real_amount、real_commission_amount、real_net_amount、settled_at、settlement_period、remark 可为 NULL

### 4. 设计索引

- **主键索引**：`pk_author_earnings` (id) - 自动创建
- **外键索引**：`idx_author_earnings_author_id` (author_id) - 用于查询作者的收益
- **外键索引**：`idx_author_earnings_transaction_id` (transaction_id) - 用于查询流水的收益记录
- **普通索引**：`idx_author_earnings_earning_type` (earning_type) - 用于按类型筛选
- **普通索引**：`idx_author_earnings_earning_status` (earning_status) - 用于按状态筛选
- **普通索引**：`idx_author_earnings_settled_at` (settled_at) - 用于按结算时间排序
- **普通索引**：`idx_author_earnings_settlement_period` (settlement_period) - 用于按结算周期查询
- **普通索引**：`idx_author_earnings_created_at` (created_at) - 用于按创建时间排序
- **普通索引**：`idx_author_earnings_virtual_currency_earning_amount` (virtual_currency_earning_amount) - 用于按虚拟货币收益金额统计和查询
- **普通索引**：`idx_author_earnings_virtual_currency_net_amount` (virtual_currency_net_amount) - 用于按虚拟货币净收益金额统计和查询
- **部分索引**：`idx_author_earnings_real_amount` (real_amount) WHERE real_amount IS NOT NULL - 用于按现实金额统计和查询（仅索引非空值）
- **部分索引**：`idx_author_earnings_real_net_amount` (real_net_amount) WHERE real_net_amount IS NOT NULL - 用于按现实净收益金额统计和查询（仅索引非空值）
- **复合索引**：`idx_author_earnings_author_status` (author_id, earning_status) - 用于查询作者的特定状态收益
- **复合索引**：`idx_author_earnings_author_settled` (author_id, settled_at) - 用于查询作者的收益并按结算时间排序
- **复合索引**：`idx_author_earnings_author_period` (author_id, settlement_period) - 用于查询作者的特定周期收益
- **复合索引**：`idx_author_earnings_author_type_amount` (author_id, earning_type, virtual_currency_net_amount) - 用于查询作者的特定类型收益并按金额排序，支持统计分析

### 5. 考虑性能优化

- 为 author_id 创建索引，支持快速查询作者的收益
- 为 transaction_id 创建索引，支持查询流水的收益记录
- 为 earning_type 和 earning_status 创建索引，支持按类型和状态筛选查询
- 为 virtual_currency_earning_amount 和 virtual_currency_net_amount 创建索引，支持按虚拟货币金额统计和查询
- 为 real_amount 和 real_net_amount 创建索引，支持按现实金额统计和查询（如果存在）
- 为 settled_at 和 settlement_period 创建索引，支持结算时间查询和周期统计
- 为 created_at 创建索引，支持时间排序查询
- 使用复合索引优化常用查询场景（如查询作者的特定状态收益、查询作者的特定周期收益、查询作者的收益并按结算时间排序）
- 使用触发器自动更新 updated_at 字段

4. 检查约束正确，确保收益类型（attendance-全勤收益、subscription-订阅收益、reading-阅读收益、collection-收藏收益、ticket-票务收益、reward-打赏收益、newbie_bonus-新手鼓励）、虚拟货币收益金额、佣金比例、收益状态等是有效的值，且收益类型与功能概述（Core.md）中的收益设置（3.2）保持一致
5. 金额字段设计正确，主要使用虚拟货币金额（消费货币）进行计算和存储，符合功能概述中收益使用消费货币计算的要求

7. 索引设计合理，支持常用查询场景（包括按作者查询、按类型查询、按状态查询、按结算周期查询、按虚拟货币金额统计、按现实金额统计）
8. 表注释和字段注释完整，使用中文，明确说明虚拟货币和现实金额的用途
9. 触发器正确创建，自动更新 updated_at 字段
10. 表设计符合 PostgreSQL 数据库设计规范
