## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解作者管理的详细需求：
- 作者资料：作者详细信息，包括昵称、头像、个人简介等
- 作者资料与作者表是一对一关系

### 2. 设计作者资料表结构

设计 `author_profiles` 表，包含作者详细资料信息：

```sql
CREATE TABLE author_profiles (
    id BIGSERIAL PRIMARY KEY,
    author_id BIGINT NOT NULL UNIQUE,
    nickname VARCHAR(100),
    avatar_url VARCHAR(500),
    bio TEXT,
    extra_data JSONB,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_author_profiles_author_id FOREIGN KEY (author_id) REFERENCES authors(id) ON DELETE CASCADE
);

-- 创建索引
CREATE INDEX idx_author_profiles_author_id ON author_profiles(author_id);
CREATE UNIQUE INDEX uk_author_profiles_author_id ON author_profiles(author_id);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_author_profiles_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_author_profiles_updated_at
    BEFORE UPDATE ON author_profiles
    FOR EACH ROW
    EXECUTE FUNCTION update_author_profiles_updated_at();

-- 添加表注释
COMMENT ON TABLE author_profiles IS '作者资料表，存储作者详细资料信息';
COMMENT ON COLUMN author_profiles.id IS '作者资料ID，自增主键';
COMMENT ON COLUMN author_profiles.author_id IS '作者ID，外键关联authors表，一对一关系';
COMMENT ON COLUMN author_profiles.nickname IS '作者昵称，用于显示';
COMMENT ON COLUMN author_profiles.avatar_url IS '头像URL，作者头像图片地址';
COMMENT ON COLUMN author_profiles.bio IS '个人简介，作者自我介绍';
COMMENT ON COLUMN author_profiles.extra_data IS '扩展数据，JSONB格式，存储额外的作者信息';
COMMENT ON COLUMN author_profiles.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN author_profiles.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_author_profiles` (id)
- **外键约束**：`fk_author_profiles_author_id` (author_id) 关联 authors(id)，级联删除
- **唯一约束**：`uk_author_profiles_author_id` (author_id) - 确保一个作者只有一条资料记录
- **非空约束**：author_id 必须非空

### 4. 设计索引

- **主键索引**：`pk_author_profiles` (id) - 自动创建
- **外键索引**：`idx_author_profiles_author_id` (author_id) - 用于快速查询作者的资料
- **唯一索引**：`uk_author_profiles_author_id` (author_id) - 确保唯一性

### 5. 考虑性能优化

- 为 author_id 创建唯一索引，支持快速查询和确保一对一关系
- 外键约束使用 CASCADE 删除，确保数据一致性
- 使用触发器自动更新 updated_at 字段
