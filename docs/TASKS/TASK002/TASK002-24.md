### 1. 分析需求文档

### 2. 设计漫画章节表结构

设计 `comic_chapters` 表，包含漫画章节基本信息：

```sql
CREATE TABLE comic_chapters (
    id BIGSERIAL PRIMARY KEY,
    comic_id BIGINT NOT NULL,
    chapter_name VARCHAR(200) NOT NULL,
    chapter_description TEXT,
    chapter_order INTEGER NOT NULL DEFAULT 0,
    image_count INTEGER DEFAULT 0,
    is_published BOOLEAN DEFAULT FALSE,
    is_vip BOOLEAN DEFAULT FALSE,
    published_at TIMESTAMPTZ,
    view_count BIGINT DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_comic_chapters_comic_id FOREIGN KEY (comic_id) REFERENCES comics(id) ON DELETE CASCADE,
    CONSTRAINT uk_comic_chapters_comic_id_chapter_order UNIQUE (comic_id, chapter_order)
);

-- 创建索引
CREATE INDEX idx_comic_chapters_comic_id ON comic_chapters(comic_id);
CREATE INDEX idx_comic_chapters_chapter_order ON comic_chapters(chapter_order);
CREATE INDEX idx_comic_chapters_is_published ON comic_chapters(is_published);
CREATE INDEX idx_comic_chapters_is_vip ON comic_chapters(is_vip);
CREATE INDEX idx_comic_chapters_published_at ON comic_chapters(published_at);
CREATE UNIQUE INDEX uk_comic_chapters_comic_id_chapter_order ON comic_chapters(comic_id, chapter_order);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_comic_chapters_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_comic_chapters_updated_at
    BEFORE UPDATE ON comic_chapters
    FOR EACH ROW
    EXECUTE FUNCTION update_comic_chapters_updated_at();

-- 添加表注释
COMMENT ON TABLE comic_chapters IS '漫画章节表，存储漫画章节信息';
COMMENT ON COLUMN comic_chapters.id IS '章节ID，自增主键';
COMMENT ON COLUMN comic_chapters.comic_id IS '漫画ID，外键关联comics表';
COMMENT ON COLUMN comic_chapters.chapter_name IS '章节名称';
COMMENT ON COLUMN comic_chapters.chapter_description IS '章节描述';
COMMENT ON COLUMN comic_chapters.chapter_order IS '章节排序，数字越小越靠前，用于排序显示';
COMMENT ON COLUMN comic_chapters.image_count IS '图片数量，章节包含的图片数量';
COMMENT ON COLUMN comic_chapters.is_published IS '是否发布，TRUE-已发布，FALSE-未发布';
COMMENT ON COLUMN comic_chapters.is_vip IS '是否VIP章节，TRUE-VIP章节需要付费，FALSE-免费章节';
COMMENT ON COLUMN comic_chapters.published_at IS '发布时间';
COMMENT ON COLUMN comic_chapters.view_count IS '浏览次数';
COMMENT ON COLUMN comic_chapters.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN comic_chapters.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_comic_chapters` (id)
- **外键约束**：`fk_comic_chapters_comic_id` (comic_id) 关联 comics(id)，级联删除
- **唯一约束**：`uk_comic_chapters_comic_id_chapter_order` (comic_id, chapter_order) - 确保同一漫画的章节排序唯一
- **非空约束**：comic_id、chapter_name 必须非空
- **默认值**：chapter_order 默认为 0，image_count 默认为 0，view_count 默认为 0，is_published、is_vip 默认为 FALSE

### 4. 设计索引

- **主键索引**：`pk_comic_chapters` (id) - 自动创建
- **外键索引**：`idx_comic_chapters_comic_id` (comic_id) - 用于查询漫画的章节
- **普通索引**：`idx_comic_chapters_chapter_order` (chapter_order) - 用于排序查询
- **普通索引**：`idx_comic_chapters_is_published` (is_published) - 用于发布状态筛选
- **普通索引**：`idx_comic_chapters_is_vip` (is_vip) - 用于VIP状态筛选
- **普通索引**：`idx_comic_chapters_published_at` (published_at) - 用于按发布时间排序
- **唯一索引**：`uk_comic_chapters_comic_id_chapter_order` (comic_id, chapter_order) - 确保唯一性

### 5. 考虑性能优化

- 为 comic_id 创建索引，支持快速查询漫画的章节
- 为 chapter_order 创建索引，支持排序查询
- 为 is_published 和 is_vip 创建索引，支持状态筛选查询
- 使用复合唯一索引，确保同一漫画的章节排序唯一
- 使用触发器自动更新 updated_at 字段
