## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解作者等级管理的详细需求：
- 作者等级管理：作者等级名称、作者等级描述、作者等级达成条件（作品数、平均评分、推荐数、投票数、评论数、打赏数等可选参数）
- 作者等级用于作者等级管理，支持按条件自动升级

### 2. 设计作者等级表结构

设计 `author_levels` 表，包含作者等级基本信息：

```sql
CREATE TABLE author_levels (
    id BIGSERIAL PRIMARY KEY,
    level_name VARCHAR(100) NOT NULL UNIQUE,
    level_description TEXT,
    level_conditions JSONB,
    level_order INTEGER NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_author_levels_level_name ON author_levels(level_name);
CREATE INDEX idx_author_levels_level_order ON author_levels(level_order);
CREATE INDEX idx_author_levels_is_active ON author_levels(is_active);
CREATE UNIQUE INDEX uk_author_levels_level_name ON author_levels(level_name);
CREATE UNIQUE INDEX uk_author_levels_level_order ON author_levels(level_order);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_author_levels_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_author_levels_updated_at
    BEFORE UPDATE ON author_levels
    FOR EACH ROW
    EXECUTE FUNCTION update_author_levels_updated_at();

-- 添加表注释
COMMENT ON TABLE author_levels IS '作者等级表，存储网站作者等级信息';
COMMENT ON COLUMN author_levels.id IS '作者等级ID，自增主键';
COMMENT ON COLUMN author_levels.level_name IS '作者等级名称，唯一标识符';
COMMENT ON COLUMN author_levels.level_description IS '作者等级描述，说明等级的用途和特权';
COMMENT ON COLUMN author_levels.level_conditions IS '作者等级达成条件，JSONB格式，包含作品数、平均评分、推荐数、投票数、评论数、打赏数等可选参数';
COMMENT ON COLUMN author_levels.level_order IS '等级排序，数字越小等级越高，用于排序和比较';
COMMENT ON COLUMN author_levels.is_active IS '是否启用，TRUE-启用，FALSE-禁用';
COMMENT ON COLUMN author_levels.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN author_levels.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_author_levels` (id)
- **唯一约束**：`uk_author_levels_level_name` (level_name)、`uk_author_levels_level_order` (level_order)
- **非空约束**：level_name、level_order 必须非空
- **默认值**：is_active 默认为 TRUE

### 4. 设计索引

- **主键索引**：`pk_author_levels` (id) - 自动创建
- **唯一索引**：`uk_author_levels_level_name` (level_name) - 确保作者等级名称唯一
- **唯一索引**：`uk_author_levels_level_order` (level_order) - 确保作者等级排序唯一
- **普通索引**：`idx_author_levels_level_name` (level_name) - 用于快速查询
- **普通索引**：`idx_author_levels_level_order` (level_order) - 用于排序查询
- **普通索引**：`idx_author_levels_is_active` (is_active) - 用于筛选启用的作者等级

### 5. 考虑性能优化

- 为 level_name 和 level_order 创建唯一索引，支持快速查询和唯一性检查
- 为 is_active 创建索引，支持状态筛选查询
- 使用触发器自动更新 updated_at 字段
