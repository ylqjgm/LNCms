## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解权限管理的详细需求：
- 用户组管理：用户组名称、用户组描述、用户组权限等
- 默认用户组：VIP用户组、普通用户组等

### 2. 创建初始化脚本文件

使用 golang-migrate 工具创建初始化脚本：
- 初始化脚本目录：`migrations/core/seeds/`
- 初始化脚本命名：`YYYYMMDDHHMMSS_seed_default_user_groups.up.sql` 和 `YYYYMMDDHHMMSS_seed_default_user_groups.down.sql`

### 3. 编写初始化脚本

#### 3.1 升级脚本（up.sql）

插入默认用户组数据：

```sql
-- 插入默认用户组
INSERT INTO user_groups (group_name, group_description, is_system, created_at) VALUES
('vip', 'VIP用户组', true, CURRENT_TIMESTAMP),
('normal', '普通用户组', true, CURRENT_TIMESTAMP)
ON CONFLICT (group_name) DO NOTHING;
```

#### 3.2 回滚脚本（down.sql）

删除默认用户组数据：

```sql
-- 删除默认用户组（仅删除系统用户组）
DELETE FROM user_groups WHERE is_system = true AND group_name IN ('vip', 'normal');
```

### 4. 初始化脚本要求

- **幂等性**：初始化脚本应该是幂等的，可以安全地重复执行
- **使用 INSERT ... ON CONFLICT**：使用 `ON CONFLICT DO NOTHING` 确保幂等性
- **可回滚**：每个升级脚本必须有对应的回滚脚本
- **数据完整性**：确保插入的数据符合表约束和业务规则

### 5. 验证初始化脚本

- 使用 golang-migrate 工具验证初始化脚本语法
- 测试升级和回滚操作
- 确保所有用户组数据都正确插入
