## 任务描述

编写默认等级的基础数据初始化脚本，使用 golang-migrate 工具创建等级表的初始化数据脚本。必须遵循 PostgreSQL 数据库设计规范。

## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解权限管理的详细需求：
- 等级管理：等级名称、等级值、等级经验值等
- 默认等级：1-5级用户等级

### 2. 创建初始化脚本文件

使用 golang-migrate 工具创建初始化脚本：
- 初始化脚本目录：`migrations/core/seeds/`
- 初始化脚本命名：`YYYYMMDDHHMMSS_seed_default_levels.up.sql` 和 `YYYYMMDDHHMMSS_seed_default_levels.down.sql`

### 3. 编写初始化脚本

#### 3.1 升级脚本（up.sql）

插入默认等级数据：

```sql
-- 插入默认等级
INSERT INTO levels (level_name, level_value, experience_required, is_system, created_at) VALUES
('新手', 1, 0, true, CURRENT_TIMESTAMP),
('初级', 2, 100, true, CURRENT_TIMESTAMP),
('中级', 3, 500, true, CURRENT_TIMESTAMP),
('高级', 4, 2000, true, CURRENT_TIMESTAMP),
('专家', 5, 10000, true, CURRENT_TIMESTAMP)
ON CONFLICT (level_value) DO NOTHING;
```

#### 3.2 回滚脚本（down.sql）

删除默认等级数据：

```sql
-- 删除默认等级（仅删除系统等级）
DELETE FROM levels WHERE is_system = true AND level_value BETWEEN 1 AND 5;
```

### 4. 初始化脚本要求

- **幂等性**：初始化脚本应该是幂等的，可以安全地重复执行
- **使用 INSERT ... ON CONFLICT**：使用 `ON CONFLICT DO NOTHING` 确保幂等性
- **可回滚**：每个升级脚本必须有对应的回滚脚本
- **数据完整性**：确保插入的数据符合表约束和业务规则

### 5. 验证初始化脚本

- 使用 golang-migrate 工具验证初始化脚本语法
- 测试升级和回滚操作
- 确保所有等级数据都正确插入

## 验收标准
