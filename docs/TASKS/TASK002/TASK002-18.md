## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解标签管理的详细需求：
- 标签管理：标签添加（标签名称、标签描述）、标签修改、标签删除
- 标签用于内容分类和检索

### 2. 设计标签表结构

设计 `tags` 表，包含标签基本信息：

```sql
CREATE TABLE tags (
    id BIGSERIAL PRIMARY KEY,
    tag_name VARCHAR(100) NOT NULL UNIQUE,
    tag_description TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE UNIQUE INDEX uk_tags_tag_name ON tags(tag_name);
CREATE INDEX idx_tags_is_active ON tags(is_active);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_tags_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_tags_updated_at
    BEFORE UPDATE ON tags
    FOR EACH ROW
    EXECUTE FUNCTION update_tags_updated_at();

-- 添加表注释
COMMENT ON TABLE tags IS '标签表，存储内容标签信息，标签为通用标签，可用于小说、漫画、音频、视频等多种内容类型';
COMMENT ON COLUMN tags.id IS '标签ID，自增主键';
COMMENT ON COLUMN tags.tag_name IS '标签名称，唯一标识符';
COMMENT ON COLUMN tags.tag_description IS '标签描述，说明标签的用途和含义';
COMMENT ON COLUMN tags.is_active IS '是否启用，TRUE-启用，FALSE-禁用';
COMMENT ON COLUMN tags.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN tags.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_tags` (id)
- **唯一约束**：`uk_tags_tag_name` (tag_name) - 确保标签名称唯一
- **非空约束**：tag_name 必须非空
- **默认值**：is_active 默认为 TRUE

### 4. 设计索引

- **主键索引**：`pk_tags` (id) - 自动创建
- **唯一索引**：`uk_tags_tag_name` (tag_name) - 确保标签名称唯一，同时支持快速查询和唯一性检查
- **普通索引**：`idx_tags_is_active` (is_active) - 用于状态筛选，查询启用的标签列表

### 5. 考虑性能优化

- 为 tag_name 创建唯一索引，支持快速查询和唯一性检查
- 为 is_active 创建索引，支持状态筛选查询，常用于查询启用的标签列表
- 使用触发器自动更新 updated_at 字段，确保更新时间准确
- 标签表作为基础数据表，数据量相对较小，索引设计以支持常用查询场景为主
