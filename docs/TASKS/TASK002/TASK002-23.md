## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解漫画管理的详细需求：
- 漫画管理：漫画信息管理，包括漫画标题、简介、封面、状态等
- 漫画属于作者，需要关联作者表
- 漫画需要支持地区，一部漫画仅支持一个地区
- 漫画需要支持连载状态（连载中、已完结、断更）
- 漫画需要支持是否上架标识，用以标注漫画是否为收费模式

### 2. 设计漫画表结构

设计 `comics` 表，包含漫画基本信息：

```sql
CREATE TABLE comics (
    id BIGSERIAL PRIMARY KEY,
    author_id BIGINT NOT NULL,
    region_id BIGINT,
    title VARCHAR(200) NOT NULL,
    subtitle VARCHAR(200),
    description TEXT,
    cover_url VARCHAR(500),
    status VARCHAR(20) NOT NULL DEFAULT 'draft',
    serial_status VARCHAR(20) NOT NULL DEFAULT 'serializing',
    is_published BOOLEAN DEFAULT FALSE,
    is_shelved BOOLEAN DEFAULT FALSE,
    published_at TIMESTAMPTZ,
    page_count INTEGER DEFAULT 0,
    view_count BIGINT DEFAULT 0,
    favorite_count BIGINT DEFAULT 0,
    rating DECIMAL(3,2) DEFAULT 0.00,
    rating_count BIGINT DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_comics_author_id FOREIGN KEY (author_id) REFERENCES authors(id) ON DELETE CASCADE,
    CONSTRAINT fk_comics_region_id FOREIGN KEY (region_id) REFERENCES regions(id) ON DELETE SET NULL,
    CONSTRAINT ck_comics_status CHECK (status IN ('draft', 'published', 'archived', 'deleted')),
    CONSTRAINT ck_comics_serial_status CHECK (serial_status IN ('serializing', 'completed', 'suspended'))
);

-- 创建索引
CREATE INDEX idx_comics_author_id ON comics(author_id);
CREATE INDEX idx_comics_region_id ON comics(region_id);
CREATE INDEX idx_comics_title ON comics(title);
CREATE INDEX idx_comics_status ON comics(status);
CREATE INDEX idx_comics_serial_status ON comics(serial_status);
CREATE INDEX idx_comics_is_published ON comics(is_published);
CREATE INDEX idx_comics_is_shelved ON comics(is_shelved);
CREATE INDEX idx_comics_published_at ON comics(published_at);
CREATE INDEX idx_comics_created_at ON comics(created_at);
CREATE INDEX idx_comics_rating ON comics(rating);
CREATE INDEX idx_comics_author_status ON comics(author_id, status);
CREATE INDEX idx_comics_region_status ON comics(region_id, status);
CREATE INDEX idx_comics_published_created ON comics(is_published, created_at);
CREATE INDEX idx_comics_shelved_published ON comics(is_shelved, is_published);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_comics_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_comics_updated_at
    BEFORE UPDATE ON comics
    FOR EACH ROW
    EXECUTE FUNCTION update_comics_updated_at();

-- 添加表注释
COMMENT ON TABLE comics IS '漫画表，存储漫画基本信息';
COMMENT ON COLUMN comics.id IS '漫画ID，自增主键';
COMMENT ON COLUMN comics.author_id IS '作者ID，外键关联authors表';
COMMENT ON COLUMN comics.region_id IS '地区ID，外键关联regions表，一部漫画仅支持一个地区';
COMMENT ON COLUMN comics.title IS '漫画标题';
COMMENT ON COLUMN comics.subtitle IS '漫画副标题';
COMMENT ON COLUMN comics.description IS '漫画简介';
COMMENT ON COLUMN comics.cover_url IS '封面URL，漫画封面图片地址';
COMMENT ON COLUMN comics.status IS '漫画状态：draft-草稿、published-已发布、archived-已归档、deleted-已删除';
COMMENT ON COLUMN comics.serial_status IS '连载状态：serializing-连载中、completed-已完结、suspended-断更';
COMMENT ON COLUMN comics.is_published IS '是否发布，TRUE-已发布，FALSE-未发布';
COMMENT ON COLUMN comics.is_shelved IS '是否上架，TRUE-已上架（收费模式），FALSE-未上架（免费模式）';
COMMENT ON COLUMN comics.published_at IS '发布时间';
COMMENT ON COLUMN comics.page_count IS '总页数';
COMMENT ON COLUMN comics.view_count IS '浏览次数';
COMMENT ON COLUMN comics.favorite_count IS '收藏次数';
COMMENT ON COLUMN comics.rating IS '评分，范围0.00-10.00';
COMMENT ON COLUMN comics.rating_count IS '评分人数';
COMMENT ON COLUMN comics.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN comics.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_comics` (id)
- **外键约束**：`fk_comics_author_id` (author_id) 关联 authors(id)，级联删除
- **外键约束**：`fk_comics_region_id` (region_id) 关联 regions(id)，删除时设置为 NULL
- **检查约束**：`ck_comics_status` (status) - 确保状态是预定义的值
- **检查约束**：`ck_comics_serial_status` (serial_status) - 确保连载状态是预定义的值
- **非空约束**：author_id、title、status、serial_status 必须非空
- **默认值**：status 默认为 'draft'，serial_status 默认为 'serializing'，is_published 默认为 FALSE，is_shelved 默认为 FALSE，page_count、view_count、favorite_count、rating、rating_count 默认为 0

### 4. 设计索引

- **主键索引**：`pk_comics` (id) - 自动创建
- **外键索引**：`idx_comics_author_id` (author_id) - 用于查询作者的漫画
- **外键索引**：`idx_comics_region_id` (region_id) - 用于查询地区的漫画
- **普通索引**：`idx_comics_title` (title) - 用于标题搜索
- **普通索引**：`idx_comics_status` (status) - 用于状态筛选
- **普通索引**：`idx_comics_serial_status` (serial_status) - 用于连载状态筛选
- **普通索引**：`idx_comics_is_published` (is_published) - 用于发布状态筛选
- **普通索引**：`idx_comics_is_shelved` (is_shelved) - 用于上架状态筛选
- **普通索引**：`idx_comics_published_at` (published_at) - 用于按发布时间排序
- **普通索引**：`idx_comics_created_at` (created_at) - 用于按创建时间排序
- **普通索引**：`idx_comics_rating` (rating) - 用于按评分排序
- **复合索引**：`idx_comics_author_status` (author_id, status) - 用于查询作者的漫画并按状态筛选
- **复合索引**：`idx_comics_region_status` (region_id, status) - 用于查询地区的漫画并按状态筛选
- **复合索引**：`idx_comics_published_created` (is_published, created_at) - 用于查询已发布的漫画并按时间排序
- **复合索引**：`idx_comics_shelved_published` (is_shelved, is_published) - 用于查询上架且已发布的漫画

### 5. 考虑性能优化

- 为 author_id 创建索引，支持快速查询作者的漫画
- 为 region_id 创建索引，支持快速查询地区的漫画
- 为 status、serial_status、is_published、is_shelved 创建索引，支持状态筛选查询
- 为 published_at 和 created_at 创建索引，支持时间排序查询
- 为 rating 创建索引，支持按评分排序
- 使用复合索引优化常用查询场景（如按地区和状态查询、按上架和发布状态查询）
- 使用触发器自动更新 updated_at 字段
