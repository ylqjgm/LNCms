## 实现指南

### 1. 分析需求文档

- 用户组管理：用户组名称、用户组描述
- 用户组用于对用户进行分组管理，支持权限分配

### 2. 设计用户组表结构

设计 `user_groups` 表，包含用户组基本信息：

```sql
CREATE TABLE user_groups (
    id BIGSERIAL PRIMARY KEY,
    group_name VARCHAR(100) NOT NULL UNIQUE,
    group_description TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_user_groups_group_name ON user_groups(group_name);
CREATE INDEX idx_user_groups_is_active ON user_groups(is_active);
CREATE UNIQUE INDEX uk_user_groups_group_name ON user_groups(group_name);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_user_groups_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_user_groups_updated_at
    BEFORE UPDATE ON user_groups
    FOR EACH ROW
    EXECUTE FUNCTION update_user_groups_updated_at();

-- 添加表注释
COMMENT ON TABLE user_groups IS '用户组表，存储网站用户组信息';
COMMENT ON COLUMN user_groups.id IS '用户组ID，自增主键';
COMMENT ON COLUMN user_groups.group_name IS '用户组名称，唯一标识符';
COMMENT ON COLUMN user_groups.group_description IS '用户组描述，说明用户组的用途和权限范围';
COMMENT ON COLUMN user_groups.is_active IS '是否启用，TRUE-启用，FALSE-禁用';
COMMENT ON COLUMN user_groups.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN user_groups.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_user_groups` (id)
- **唯一约束**：`uk_user_groups_group_name` (group_name)
- **非空约束**：group_name 必须非空
- **默认值**：is_active 默认为 TRUE

### 4. 设计索引

- **主键索引**：`pk_user_groups` (id) - 自动创建
- **唯一索引**：`uk_user_groups_group_name` (group_name) - 确保用户组名称唯一
- **普通索引**：`idx_user_groups_group_name` (group_name) - 用于快速查询
- **普通索引**：`idx_user_groups_is_active` (is_active) - 用于筛选启用的用户组

### 5. 考虑性能优化

- 为 group_name 创建唯一索引，支持快速查询和唯一性检查
- 为 is_active 创建索引，支持状态筛选查询
- 使用触发器自动更新 updated_at 字段
