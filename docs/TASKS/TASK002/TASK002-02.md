## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解角色管理的详细需求：
- 角色管理：角色名称、角色描述
- 角色仅用于后台管理，主要用于后续的组织架构部分进行角色划分（如部长、组长等）

### 2. 设计角色表结构

设计 `roles` 表，包含角色基本信息：

```sql
CREATE TABLE roles (
    id BIGSERIAL PRIMARY KEY,
    role_name VARCHAR(100) NOT NULL UNIQUE,
    role_description TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    is_system BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_roles_role_name ON roles(role_name);
CREATE INDEX idx_roles_is_active ON roles(is_active);
CREATE INDEX idx_roles_is_system ON roles(is_system);
CREATE UNIQUE INDEX uk_roles_role_name ON roles(role_name);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_roles_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_roles_updated_at
    BEFORE UPDATE ON roles
    FOR EACH ROW
    EXECUTE FUNCTION update_roles_updated_at();

-- 添加表注释
COMMENT ON TABLE roles IS '角色表，存储系统管理角色信息';
COMMENT ON COLUMN roles.id IS '角色ID，自增主键';
COMMENT ON COLUMN roles.role_name IS '角色名称，唯一标识符，用于后台管理中的角色划分，如部长、组长、编辑等';
COMMENT ON COLUMN roles.role_description IS '角色描述，说明角色的用途和权限范围';
COMMENT ON COLUMN roles.is_active IS '是否启用，TRUE-启用，FALSE-禁用';
COMMENT ON COLUMN roles.is_system IS '是否为系统角色，TRUE-系统角色不允许删除，FALSE-普通角色可以删除';
COMMENT ON COLUMN roles.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN roles.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_roles` (id)
- **唯一约束**：`uk_roles_role_name` (role_name)
- **非空约束**：role_name 必须非空
- **默认值**：is_active 默认为 TRUE，is_system 默认为 FALSE

### 4. 设计索引

- **主键索引**：`pk_roles` (id) - 自动创建
- **唯一索引**：`uk_roles_role_name` (role_name) - 确保角色名称唯一
- **普通索引**：`idx_roles_role_name` (role_name) - 用于快速查询
- **普通索引**：`idx_roles_is_active` (is_active) - 用于筛选启用的角色
- **普通索引**：`idx_roles_is_system` (is_system) - 用于筛选系统角色

### 5. 考虑性能优化

- 为 role_name 创建唯一索引，支持快速查询和唯一性检查
- 为 is_active 和 is_system 创建索引，支持状态筛选查询
- 使用触发器自动更新 updated_at 字段
