## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解漫画管理的详细需求：
- 漫画章节图片管理：章节图片，包括图片文件、图片排序等
- 图片属于章节，需要关联章节表

### 2. 设计漫画章节图片表结构

设计 `comic_chapter_images` 表，包含漫画章节图片信息：

```sql
CREATE TABLE comic_chapter_images (
    id BIGSERIAL PRIMARY KEY,
    chapter_id BIGINT NOT NULL,
    file_id BIGINT NOT NULL,
    image_order INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_comic_chapter_images_chapter_id FOREIGN KEY (chapter_id) REFERENCES comic_chapters(id) ON DELETE CASCADE,
    CONSTRAINT fk_comic_chapter_images_file_id FOREIGN KEY (file_id) REFERENCES files(id) ON DELETE CASCADE,
    CONSTRAINT uk_comic_chapter_images_chapter_id_image_order UNIQUE (chapter_id, image_order)
);

-- 创建索引
CREATE INDEX idx_comic_chapter_images_chapter_id ON comic_chapter_images(chapter_id);
CREATE INDEX idx_comic_chapter_images_file_id ON comic_chapter_images(file_id);
CREATE INDEX idx_comic_chapter_images_image_order ON comic_chapter_images(image_order);
CREATE UNIQUE INDEX uk_comic_chapter_images_chapter_id_image_order ON comic_chapter_images(chapter_id, image_order);

-- 添加表注释
COMMENT ON TABLE comic_chapter_images IS '漫画章节图片表，存储漫画章节图片信息';
COMMENT ON COLUMN comic_chapter_images.id IS '图片ID，自增主键';
COMMENT ON COLUMN comic_chapter_images.chapter_id IS '章节ID，外键关联comic_chapters表';
COMMENT ON COLUMN comic_chapter_images.file_id IS '文件ID，外键关联files表（存储模块）';
COMMENT ON COLUMN comic_chapter_images.image_order IS '图片排序，数字越小越靠前，用于排序显示';
COMMENT ON COLUMN comic_chapter_images.created_at IS '创建时间，自动设置为当前时间';
```

### 3. 设计约束

- **主键约束**：`pk_comic_chapter_images` (id)
- **外键约束**：`fk_comic_chapter_images_chapter_id` (chapter_id) 关联 comic_chapters(id)，级联删除
- **外键约束**：`fk_comic_chapter_images_file_id` (file_id) 关联 files(id)，级联删除（files 表在存储模块）
- **唯一约束**：`uk_comic_chapter_images_chapter_id_image_order` (chapter_id, image_order) - 确保同一章节的图片排序唯一
- **非空约束**：chapter_id、file_id 必须非空
- **默认值**：image_order 默认为 0

### 4. 设计索引

- **主键索引**：`pk_comic_chapter_images` (id) - 自动创建
- **外键索引**：`idx_comic_chapter_images_chapter_id` (chapter_id) - 用于查询章节的图片
- **外键索引**：`idx_comic_chapter_images_file_id` (file_id) - 用于关联查询
- **普通索引**：`idx_comic_chapter_images_image_order` (image_order) - 用于排序查询
- **唯一索引**：`uk_comic_chapter_images_chapter_id_image_order` (chapter_id, image_order) - 确保唯一性

### 5. 考虑性能优化

- 为 chapter_id 创建索引，支持快速查询章节的图片
- 为 file_id 创建索引，支持关联查询
- 使用复合唯一索引，确保同一章节的图片排序唯一
- 外键约束使用 CASCADE 删除，确保数据一致性
