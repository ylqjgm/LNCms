## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解权限管理的详细需求：
- 权限管理：权限名称、权限目标
- 权限用于对各分组、等级进行权限管理，支持细粒度权限控制

### 2. 设计权限表结构

设计 `permissions` 表，包含权限基本信息：

```sql
CREATE TABLE permissions (
    id BIGSERIAL PRIMARY KEY,
    permission_name VARCHAR(100) NOT NULL UNIQUE,
    permission_target VARCHAR(200),
    permission_description TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_permissions_permission_name ON permissions(permission_name);
CREATE INDEX idx_permissions_permission_target ON permissions(permission_target);
CREATE INDEX idx_permissions_is_active ON permissions(is_active);
CREATE UNIQUE INDEX uk_permissions_permission_name ON permissions(permission_name);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_permissions_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_permissions_updated_at
    BEFORE UPDATE ON permissions
    FOR EACH ROW
    EXECUTE FUNCTION update_permissions_updated_at();

-- 添加表注释
COMMENT ON TABLE permissions IS '权限表，存储系统权限信息';
COMMENT ON COLUMN permissions.id IS '权限ID，自增主键';
COMMENT ON COLUMN permissions.permission_name IS '权限名称，唯一标识符，如 user.create、article.update 等';
COMMENT ON COLUMN permissions.permission_target IS '权限目标，说明权限作用的目标资源或操作';
COMMENT ON COLUMN permissions.permission_description IS '权限描述，说明权限的用途和范围';
COMMENT ON COLUMN permissions.is_active IS '是否启用，TRUE-启用，FALSE-禁用';
COMMENT ON COLUMN permissions.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN permissions.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_permissions` (id)
- **唯一约束**：`uk_permissions_permission_name` (permission_name)
- **非空约束**：permission_name 必须非空
- **默认值**：is_active 默认为 TRUE

### 4. 设计索引

- **主键索引**：`pk_permissions` (id) - 自动创建
- **唯一索引**：`uk_permissions_permission_name` (permission_name) - 确保权限名称唯一
- **普通索引**：`idx_permissions_permission_name` (permission_name) - 用于快速查询
- **普通索引**：`idx_permissions_permission_target` (permission_target) - 用于按目标查询
- **普通索引**：`idx_permissions_is_active` (is_active) - 用于筛选启用的权限

### 5. 考虑性能优化

- 为 permission_name 创建唯一索引，支持快速查询和唯一性检查
- 为 permission_target 创建索引，支持按目标筛选查询
- 为 is_active 创建索引，支持状态筛选查询
- 使用触发器自动更新 updated_at 字段
