## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解用户管理的详细需求：
- 用户资料：用户详细信息，包括昵称、头像、性别、生日、个人简介等
- 用户资料与用户表是一对一关系

### 2. 设计用户资料表结构

设计 `user_profiles` 表，包含用户详细资料信息：

```sql
CREATE TABLE user_profiles (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL UNIQUE,
    nickname VARCHAR(100),
    avatar_url VARCHAR(500),
    gender VARCHAR(10),
    birthday DATE,
    bio TEXT,
    extra_data JSONB,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_user_profiles_user_id FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- 创建索引
CREATE INDEX idx_user_profiles_user_id ON user_profiles(user_id);
CREATE UNIQUE INDEX uk_user_profiles_user_id ON user_profiles(user_id);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_user_profiles_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_user_profiles_updated_at
    BEFORE UPDATE ON user_profiles
    FOR EACH ROW
    EXECUTE FUNCTION update_user_profiles_updated_at();

-- 添加表注释
COMMENT ON TABLE user_profiles IS '用户资料表，存储用户详细资料信息';
COMMENT ON COLUMN user_profiles.id IS '用户资料ID，自增主键';
COMMENT ON COLUMN user_profiles.user_id IS '用户ID，外键关联users表，一对一关系';
COMMENT ON COLUMN user_profiles.nickname IS '用户昵称，用于显示';
COMMENT ON COLUMN user_profiles.avatar_url IS '头像URL，用户头像图片地址';
COMMENT ON COLUMN user_profiles.gender IS '性别，可选值：male-男、female-女、other-其他';
COMMENT ON COLUMN user_profiles.birthday IS '生日，日期格式';
COMMENT ON COLUMN user_profiles.bio IS '个人简介，用户自我介绍';
COMMENT ON COLUMN user_profiles.extra_data IS '扩展数据，JSONB格式，存储额外的用户信息';
COMMENT ON COLUMN user_profiles.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN user_profiles.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_user_profiles` (id)
- **外键约束**：`fk_user_profiles_user_id` (user_id) 关联 users(id)，级联删除
- **唯一约束**：`uk_user_profiles_user_id` (user_id) - 确保一个用户只有一条资料记录
- **非空约束**：user_id 必须非空

### 4. 设计索引

- **主键索引**：`pk_user_profiles` (id) - 自动创建
- **外键索引**：`idx_user_profiles_user_id` (user_id) - 用于快速查询用户的资料
- **唯一索引**：`uk_user_profiles_user_id` (user_id) - 确保唯一性

### 5. 考虑性能优化

- 为 user_id 创建唯一索引，支持快速查询和确保一对一关系
- 外键约束使用 CASCADE 删除，确保数据一致性
- 使用触发器自动更新 updated_at 字段
