## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解地区管理的详细需求：
- 地区管理：地区添加（地区名称、上级地区、地区描述、地区排序）、地区修改、地区删除
- 地区支持树形结构，支持多级地区（国家、省、市、区等）

### 2. 设计地区表结构

设计 `regions` 表，包含地区基本信息，支持树形结构：

```sql
CREATE TABLE regions (
    id BIGSERIAL PRIMARY KEY,
    region_name VARCHAR(100) NOT NULL,
    region_description TEXT,
    parent_id BIGINT,
    region_order INTEGER NOT NULL DEFAULT 0,
    region_code VARCHAR(50),
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_regions_parent_id FOREIGN KEY (parent_id) REFERENCES regions(id) ON DELETE SET NULL
);

-- 创建索引
CREATE INDEX idx_regions_region_name ON regions(region_name);
CREATE INDEX idx_regions_parent_id ON regions(parent_id);
CREATE INDEX idx_regions_region_order ON regions(region_order);
CREATE INDEX idx_regions_region_code ON regions(region_code);
CREATE INDEX idx_regions_is_active ON regions(is_active);
CREATE INDEX idx_regions_parent_order ON regions(parent_id, region_order);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_regions_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_regions_updated_at
    BEFORE UPDATE ON regions
    FOR EACH ROW
    EXECUTE FUNCTION update_regions_updated_at();

-- 添加表注释
COMMENT ON TABLE regions IS '地区表，存储地区信息，支持树形结构';
COMMENT ON COLUMN regions.id IS '地区ID，自增主键';
COMMENT ON COLUMN regions.region_name IS '地区名称';
COMMENT ON COLUMN regions.region_description IS '地区描述，说明地区的范围和特点';
COMMENT ON COLUMN regions.parent_id IS '上级地区ID，外键关联regions表，支持树形结构，NULL表示顶级地区';
COMMENT ON COLUMN regions.region_order IS '地区排序，数字越小越靠前，用于排序显示';
COMMENT ON COLUMN regions.region_code IS '地区代码，如行政区划代码，用于标识和查询';
COMMENT ON COLUMN regions.is_active IS '是否启用，TRUE-启用，FALSE-禁用';
COMMENT ON COLUMN regions.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN regions.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_regions` (id)
- **外键约束**：`fk_regions_parent_id` (parent_id) 关联 regions(id)，删除时设置为 NULL（自引用）
- **非空约束**：region_name 必须非空
- **默认值**：region_order 默认为 0，is_active 默认为 TRUE

### 4. 设计索引

- **主键索引**：`pk_regions` (id) - 自动创建
- **外键索引**：`idx_regions_parent_id` (parent_id) - 用于查询子地区
- **普通索引**：`idx_regions_region_name` (region_name) - 用于快速查询
- **普通索引**：`idx_regions_region_order` (region_order) - 用于排序查询
- **普通索引**：`idx_regions_region_code` (region_code) - 用于按代码查询
- **普通索引**：`idx_regions_is_active` (is_active) - 用于状态筛选
- **复合索引**：`idx_regions_parent_order` (parent_id, region_order) - 用于按父地区和排序查询

### 5. 考虑性能优化

- 为 parent_id 创建索引，支持树形结构查询
- 为 region_code 创建索引，支持按代码查询
- 使用复合索引优化按父地区和排序的查询
- 使用触发器自动更新 updated_at 字段
