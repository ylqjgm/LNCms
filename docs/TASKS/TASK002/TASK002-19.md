## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解小说管理的详细需求：
- 小说管理：小说信息管理，包括小说标题、简介、封面、状态等
- 小说属于作者，需要关联作者表
- 小说需要支持分类，一部小说只支持一个分类
- 小说需要支持连载状态（连载中、已完结、断更）
- 小说需要支持是否上架标识，用以标注小说是否为收费模式

### 2. 设计小说表结构

设计 `novels` 表，包含小说基本信息：

```sql
CREATE TABLE novels (
    id BIGSERIAL PRIMARY KEY,
    author_id BIGINT NOT NULL,
    category_id BIGINT,
    title VARCHAR(200) NOT NULL,
    subtitle VARCHAR(200),
    description TEXT,
    cover_url VARCHAR(500),
    status VARCHAR(20) NOT NULL DEFAULT 'draft',
    serial_status VARCHAR(20) NOT NULL DEFAULT 'serializing',
    is_published BOOLEAN DEFAULT FALSE,
    is_shelved BOOLEAN DEFAULT FALSE,
    published_at TIMESTAMPTZ,
    word_count BIGINT DEFAULT 0,
    view_count BIGINT DEFAULT 0,
    favorite_count BIGINT DEFAULT 0,
    rating DECIMAL(3,2) DEFAULT 0.00,
    rating_count BIGINT DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_novels_author_id FOREIGN KEY (author_id) REFERENCES authors(id) ON DELETE CASCADE,
    CONSTRAINT fk_novels_category_id FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE SET NULL,
    CONSTRAINT ck_novels_status CHECK (status IN ('draft', 'published', 'archived', 'deleted')),
    CONSTRAINT ck_novels_serial_status CHECK (serial_status IN ('serializing', 'completed', 'suspended'))
);

-- 创建索引
CREATE INDEX idx_novels_author_id ON novels(author_id);
CREATE INDEX idx_novels_category_id ON novels(category_id);
CREATE INDEX idx_novels_title ON novels(title);
CREATE INDEX idx_novels_status ON novels(status);
CREATE INDEX idx_novels_serial_status ON novels(serial_status);
CREATE INDEX idx_novels_is_published ON novels(is_published);
CREATE INDEX idx_novels_is_shelved ON novels(is_shelved);
CREATE INDEX idx_novels_published_at ON novels(published_at);
CREATE INDEX idx_novels_created_at ON novels(created_at);
CREATE INDEX idx_novels_rating ON novels(rating);
CREATE INDEX idx_novels_author_status ON novels(author_id, status);
CREATE INDEX idx_novels_category_status ON novels(category_id, status);
CREATE INDEX idx_novels_published_created ON novels(is_published, created_at);
CREATE INDEX idx_novels_shelved_published ON novels(is_shelved, is_published);

-- 创建触发器自动更新 updated_at
CREATE OR REPLACE FUNCTION update_novels_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER update_novels_updated_at
    BEFORE UPDATE ON novels
    FOR EACH ROW
    EXECUTE FUNCTION update_novels_updated_at();

-- 添加表注释
COMMENT ON TABLE novels IS '小说表，存储小说基本信息';
COMMENT ON COLUMN novels.id IS '小说ID，自增主键';
COMMENT ON COLUMN novels.author_id IS '作者ID，外键关联authors表';
COMMENT ON COLUMN novels.category_id IS '分类ID，外键关联categories表，一部小说只支持一个分类';
COMMENT ON COLUMN novels.title IS '小说标题';
COMMENT ON COLUMN novels.subtitle IS '小说副标题';
COMMENT ON COLUMN novels.description IS '小说简介';
COMMENT ON COLUMN novels.cover_url IS '封面URL，小说封面图片地址';
COMMENT ON COLUMN novels.status IS '小说状态：draft-草稿、published-已发布、archived-已归档、deleted-已删除';
COMMENT ON COLUMN novels.serial_status IS '连载状态：serializing-连载中、completed-已完结、suspended-断更';
COMMENT ON COLUMN novels.is_published IS '是否发布，TRUE-已发布，FALSE-未发布';
COMMENT ON COLUMN novels.is_shelved IS '是否上架，TRUE-已上架（收费模式），FALSE-未上架（免费模式）';
COMMENT ON COLUMN novels.published_at IS '发布时间';
COMMENT ON COLUMN novels.word_count IS '总字数';
COMMENT ON COLUMN novels.view_count IS '浏览次数';
COMMENT ON COLUMN novels.favorite_count IS '收藏次数';
COMMENT ON COLUMN novels.rating IS '评分，范围0.00-10.00';
COMMENT ON COLUMN novels.rating_count IS '评分人数';
COMMENT ON COLUMN novels.created_at IS '创建时间，自动设置为当前时间';
COMMENT ON COLUMN novels.updated_at IS '更新时间，每次更新时自动更新';
```

### 3. 设计约束

- **主键约束**：`pk_novels` (id)
- **外键约束**：`fk_novels_author_id` (author_id) 关联 authors(id)，级联删除
- **外键约束**：`fk_novels_category_id` (category_id) 关联 categories(id)，删除时设置为 NULL
- **检查约束**：`ck_novels_status` (status) - 确保状态是预定义的值
- **检查约束**：`ck_novels_serial_status` (serial_status) - 确保连载状态是预定义的值
- **非空约束**：author_id、title、status、serial_status 必须非空
- **默认值**：status 默认为 'draft'，serial_status 默认为 'serializing'，is_published 默认为 FALSE，is_shelved 默认为 FALSE，word_count、view_count、favorite_count、rating、rating_count 默认为 0

### 4. 设计索引

- **主键索引**：`pk_novels` (id) - 自动创建
- **外键索引**：`idx_novels_author_id` (author_id) - 用于查询作者的小说
- **外键索引**：`idx_novels_category_id` (category_id) - 用于查询分类的小说
- **普通索引**：`idx_novels_title` (title) - 用于标题搜索
- **普通索引**：`idx_novels_status` (status) - 用于状态筛选
- **普通索引**：`idx_novels_serial_status` (serial_status) - 用于连载状态筛选
- **普通索引**：`idx_novels_is_published` (is_published) - 用于发布状态筛选
- **普通索引**：`idx_novels_is_shelved` (is_shelved) - 用于上架状态筛选
- **普通索引**：`idx_novels_published_at` (published_at) - 用于按发布时间排序
- **普通索引**：`idx_novels_created_at` (created_at) - 用于按创建时间排序
- **普通索引**：`idx_novels_rating` (rating) - 用于按评分排序
- **复合索引**：`idx_novels_author_status` (author_id, status) - 用于查询作者的小说并按状态筛选
- **复合索引**：`idx_novels_category_status` (category_id, status) - 用于查询分类的小说并按状态筛选
- **复合索引**：`idx_novels_published_created` (is_published, created_at) - 用于查询已发布的小说并按时间排序
- **复合索引**：`idx_novels_shelved_published` (is_shelved, is_published) - 用于查询上架且已发布的小说

### 5. 考虑性能优化

- 为 author_id 创建索引，支持快速查询作者的小说
- 为 category_id 创建索引，支持快速查询分类的小说
- 为 status、serial_status、is_published、is_shelved 创建索引，支持状态筛选查询
- 为 published_at 和 created_at 创建索引，支持时间排序查询
- 为 rating 创建索引，支持按评分排序
- 使用复合索引优化常用查询场景（如按分类和状态查询、按上架和发布状态查询）
- 使用触发器自动更新 updated_at 字段

9. 小说支持分类关联，一部小说只支持一个分类
