## 任务描述

设计核心模块的用户组权限关联表（user_group_permissions），用于存储用户组和权限的多对多关系。必须遵循 PostgreSQL 数据库设计规范，包含完整的字段定义、约束、索引、注释等。

TASK002-03（用户组表）、TASK002-07（权限表）

## 实现指南

### 1. 分析需求文档

阅读 `docs/Requirements/Core.md` 文档，了解权限管理的详细需求：
- 用户组权限关联：用户组和权限是多对多关系，一个用户组可以有多个权限，一个权限可以分配给多个用户组
- 用于实现基于用户组的权限管理

### 2. 设计用户组权限关联表结构

设计 `user_group_permissions` 表，包含用户组和权限的关联关系：

```sql
CREATE TABLE user_group_permissions (
    id BIGSERIAL PRIMARY KEY,
    user_group_id BIGINT NOT NULL,
    permission_id BIGINT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_user_group_permissions_user_group_id FOREIGN KEY (user_group_id) REFERENCES user_groups(id) ON DELETE CASCADE,
    CONSTRAINT fk_user_group_permissions_permission_id FOREIGN KEY (permission_id) REFERENCES permissions(id) ON DELETE CASCADE,
    CONSTRAINT uk_user_group_permissions_user_group_id_permission_id UNIQUE (user_group_id, permission_id)
);

-- 创建索引
CREATE INDEX idx_user_group_permissions_user_group_id ON user_group_permissions(user_group_id);
CREATE INDEX idx_user_group_permissions_permission_id ON user_group_permissions(permission_id);
CREATE UNIQUE INDEX uk_user_group_permissions_user_group_id_permission_id ON user_group_permissions(user_group_id, permission_id);

-- 添加表注释
COMMENT ON TABLE user_group_permissions IS '用户组权限关联表，存储用户组和权限的多对多关系';
COMMENT ON COLUMN user_group_permissions.id IS '关联ID，自增主键';
COMMENT ON COLUMN user_group_permissions.user_group_id IS '用户组ID，外键关联user_groups表';
COMMENT ON COLUMN user_group_permissions.permission_id IS '权限ID，外键关联permissions表';
COMMENT ON COLUMN user_group_permissions.created_at IS '创建时间，自动设置为当前时间';
```

### 3. 设计约束

- **主键约束**：`pk_user_group_permissions` (id)
- **外键约束**：`fk_user_group_permissions_user_group_id` (user_group_id) 关联 user_groups(id)，级联删除
- **外键约束**：`fk_user_group_permissions_permission_id` (permission_id) 关联 permissions(id)，级联删除
- **唯一约束**：`uk_user_group_permissions_user_group_id_permission_id` (user_group_id, permission_id) - 确保同一用户组和权限的组合唯一
- **非空约束**：user_group_id、permission_id 必须非空

### 4. 设计索引

- **主键索引**：`pk_user_group_permissions` (id) - 自动创建
- **外键索引**：`idx_user_group_permissions_user_group_id` (user_group_id) - 用于快速查询用户组的权限
- **外键索引**：`idx_user_group_permissions_permission_id` (permission_id) - 用于快速查询权限的用户组
- **唯一索引**：`uk_user_group_permissions_user_group_id_permission_id` (user_group_id, permission_id) - 确保唯一性

### 5. 考虑性能优化

- 为 user_group_id 和 permission_id 创建索引，支持快速查询
- 使用复合唯一索引，确保同一用户组和权限的组合唯一
- 外键约束使用 CASCADE 删除，确保数据一致性
