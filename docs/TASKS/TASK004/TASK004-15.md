### 2. 创建Level模型

在 `internal/pkg/model/level.go` 文件中创建Level结构体：

```go
package model

import (
    "database/sql/driver"
    "encoding/json"
    "time"
    "gorm.io/gorm"
)

// LevelConditions 等级达成条件结构
type LevelConditions struct {
    // ConditionType 条件类型：points-积分、currency-货币、recharge_count-充值数、consume_count-消费数、comment_count-评论数、read_count-阅读数
    ConditionType string `json:"condition_type"`
    
    // ConditionValue 条件值
    ConditionValue int64 `json:"condition_value"`
}

// Value 实现driver.Valuer接口，用于数据库存储
func (lc LevelConditions) Value() (driver.Value, error) {
    return json.Marshal(lc)
}

// Scan 实现sql.Scanner接口，用于数据库读取
func (lc *LevelConditions) Scan(value interface{}) error {
    if value == nil {
        return nil
    }
    bytes, ok := value.([]byte)
    if !ok {
        return fmt.Errorf("无法将 %T 转换为 []byte", value)
    }
    return json.Unmarshal(bytes, lc)
}

// Level 等级模型
// 用于存储用户等级信息，包括等级名称、描述和达成条件
// 表名: levels
// 索引: pk_levels (id), idx_levels_name (name)
type Level struct {
    // ID 等级ID，自增主键
    ID int64 `gorm:"primaryKey;autoIncrement" json:"id"`
    
    // Name 等级名称，用于标识等级
    Name string `gorm:"type:varchar(100);not null;index:idx_levels_name" json:"name"`
    
    // Description 等级描述，说明等级的用途和特点
    Description *string `gorm:"type:text" json:"description,omitempty"`
    
    // Conditions 等级达成条件，使用JSONB类型存储
    // 存储格式：{"condition_type": "points", "condition_value": 1000}
    Conditions LevelConditions `gorm:"type:jsonb" json:"conditions"`
    
    // CreatedAt 创建时间，自动设置为当前时间
    CreatedAt time.Time `gorm:"type:timestamptz;default:current_timestamp" json:"created_at"`
    
    // UpdatedAt 更新时间，自动设置为当前时间
    UpdatedAt time.Time `gorm:"type:timestamptz;default:current_timestamp" json:"updated_at"`
    
    // DeletedAt 软删除时间（GORM软删除支持）
    DeletedAt gorm.DeletedAt `gorm:"index" json:"deleted_at,omitempty"`
}

// TableName 指定表名
func (Level) TableName() string {
    return "levels"
}
```

### 3. 验证模型定义

确保模型定义符合以下要求：

- 主键字段使用 `bigserial` 类型（GORM会自动映射为int64）
- `name` 字段使用 `varchar(100)` 类型，设置普通索引
- `description` 字段使用 `text` 类型，允许为空
- `conditions` 字段使用 `jsonb` 类型，存储等级达成条件
- 时间字段使用 `timestamptz` 类型，支持时区
- 所有字段都有适当的GORM标签和JSON标签
- 模型实现了 `TableName()` 方法，明确指定表名
- 支持GORM软删除功能（DeletedAt字段）
- LevelConditions结构体实现了driver.Valuer和sql.Scanner接口
