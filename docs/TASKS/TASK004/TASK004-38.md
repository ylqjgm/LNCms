### 2. 创建Permission模型

在 `internal/pkg/model/permission.go` 文件中创建Permission结构体：

```go
package model

import (
    "time"
    "gorm.io/gorm"
)

// Permission 权限模型
// 用于存储系统权限信息，支持细粒度权限控制
// 表名: permissions
// 索引: pk_permissions (id), uk_permissions_permission_name (permission_name), idx_permissions_permission_target (permission_target), idx_permissions_is_active (is_active)
type Permission struct {
    // ID 权限ID，自增主键
    ID int64 `gorm:"primaryKey;autoIncrement" json:"id"`
    
    // PermissionName 权限名称，唯一标识符
    PermissionName string `gorm:"type:varchar(100);not null;uniqueIndex:uk_permissions_permission_name" json:"permission_name"`
    
    // PermissionTarget 权限目标，用于标识权限作用的对象
    PermissionTarget *string `gorm:"type:varchar(200);index:idx_permissions_permission_target" json:"permission_target,omitempty"`
    
    // PermissionDescription 权限描述，说明权限的用途和范围
    PermissionDescription *string `gorm:"type:text" json:"permission_description,omitempty"`
    
    // IsActive 是否启用，TRUE-启用，FALSE-禁用
    IsActive bool `gorm:"type:boolean;default:true;index:idx_permissions_is_active" json:"is_active"`
    
    // CreatedAt 创建时间，自动设置为当前时间
    CreatedAt time.Time `gorm:"type:timestamptz;default:current_timestamp" json:"created_at"`
    
    // UpdatedAt 更新时间，自动设置为当前时间
    UpdatedAt time.Time `gorm:"type:timestamptz;default:current_timestamp" json:"updated_at"`
    
    // DeletedAt 软删除时间（GORM软删除支持）
    DeletedAt gorm.DeletedAt `gorm:"index" json:"deleted_at,omitempty"`
}

// TableName 指定表名
func (Permission) TableName() string {
    return "permissions"
}
```

### 3. 验证模型定义

确保模型定义符合以下要求：

- 主键字段使用 `bigserial` 类型（GORM会自动映射为int64）
- `permission_name` 字段使用 `varchar(100)` 类型，设置唯一索引
- `permission_target` 字段使用 `varchar(200)` 类型，允许为空，设置索引
- `permission_description` 字段使用 `text` 类型，允许为空
- `is_active` 字段使用 `boolean` 类型，设置默认值和索引
- 时间字段使用 `timestamptz` 类型，支持时区
- 所有字段都有适当的GORM标签和JSON标签
- 模型实现了 `TableName()` 方法，明确指定表名
- 支持GORM软删除功能（DeletedAt字段）
