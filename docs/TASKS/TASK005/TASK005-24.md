### 2. 实现BanUser方法

```go
// BanUser 封禁用户
// @Summary 封禁用户
// @Description 封禁用户，设置is_locked为true
// @Tags 用户管理
// @Accept json
// @Produce json
// @Security BearerAuth
// @Param id path int true "用户ID"
// @Param request body dto.BanUserRequest true "封禁用户请求" default({"reason":"违规行为"})
// @Success 200 {object} response.Envelope{data=dto.UserResponse}
// @Failure 400 {object} response.Envelope
// @Failure 401 {object} response.Envelope
// @Failure 404 {object} response.Envelope
// @Failure 500 {object} response.Envelope
// @Router /api/v1/users/{id}/ban [put]
func (h *userHandler) BanUser(c *gin.Context) {
    // 1. 获取路径参数
    idStr := c.Param("id")
    id, err := strconv.ParseInt(idStr, 10, 64)
    if err != nil {
        response.BadRequest(c, "无效的用户ID", err)
        return
    }
    
    // 2. 绑定请求参数（可选）
    var req dto.BanUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        // 如果请求体为空，使用默认值
        req.Reason = "管理员封禁"
    }
    
    // 3. 调用UserService.BanUser
    user, err := h.userService.BanUser(c.Request.Context(), id, req.Reason)
    if err != nil {
        if isNotFoundError(err) {
            response.NotFound(c, "用户不存在", err)
            return
        }
        if isValidationError(err) {
            response.UnprocessableEntity(c, "验证失败", err)
            return
        }
        response.InternalServerError(c, "封禁用户失败", err)
        return
    }
    
    // 4. 转换为响应DTO
    userResponse := dto.UserResponseFromModel(user)
    
    // 5. 格式化响应（Envelope格式）
    response.OK(c, userResponse, "封禁用户成功")
}
```

### 6. 注意事项

- **请求参数绑定**：BanUser方法支持可选的请求体（封禁原因），UnbanUser方法不需要请求体
- **路径参数解析**：使用c.Param获取路径参数，并转换为int64类型
- **参数验证**：封禁原因可选，最大长度为500字符
- **错误处理**：区分不同的错误类型（未找到、验证失败、服务器错误）
- **响应格式**：使用Envelope格式，HTTP状态码200 OK
- **权限控制**：需要认证和管理员权限，在路由配置中应用JWT认证中间件和权限中间件
- **封禁逻辑**：BanUser设置is_locked为true，UnbanUser设置is_locked为false
- **封禁原因**：封禁时可以记录原因（可选），便于后续查看
