### 2. 实现DeleteUser方法

```go
// DeleteUser 删除用户
// @Summary 删除用户
// @Description 删除用户（物理删除或软删除，根据业务需求）
// @Tags 用户管理
// @Accept json
// @Produce json
// @Security BearerAuth
// @Param id path int true "用户ID"
// @Success 200 {object} response.Envelope
// @Failure 400 {object} response.Envelope
// @Failure 401 {object} response.Envelope
// @Failure 404 {object} response.Envelope
// @Failure 500 {object} response.Envelope
// @Router /api/v1/users/{id} [delete]
func (h *userHandler) DeleteUser(c *gin.Context) {
    // 1. 获取路径参数
    idStr := c.Param("id")
    id, err := strconv.ParseInt(idStr, 10, 64)
    if err != nil {
        response.BadRequest(c, "无效的用户ID", err)
        return
    }
    
    // 2. 调用UserService.DeleteUser
    err = h.userService.DeleteUser(c.Request.Context(), id)
    if err != nil {
        if isNotFoundError(err) {
            response.NotFound(c, "用户不存在", err)
            return
        }
        if isValidationError(err) {
            response.UnprocessableEntity(c, "验证失败", err)
            return
        }
        response.InternalServerError(c, "删除用户失败", err)
        return
    }
    
    // 3. 格式化响应（Envelope格式）
    response.OK(c, nil, "删除用户成功")
}
```

### 4. 注意事项

- **路径参数解析**：使用c.Param获取路径参数，并转换为int64类型
- **HTTP方法**：使用DELETE方法，遵循RESTful规范
- **错误处理**：区分不同的错误类型（未找到、验证失败、服务器错误）
- **响应格式**：使用Envelope格式，HTTP状态码200 OK（删除成功通常返回200，也可以返回204 No Content）
- **权限控制**：需要认证和管理员权限，在路由配置中应用JWT认证中间件和权限中间件
- **删除策略**：根据业务需求选择物理删除或软删除
- **级联删除**：考虑用户关联数据的处理（如用户会话、登录日志等）
- **安全检查**：删除操作前可以检查用户是否可以被删除（如是否有关联数据）
