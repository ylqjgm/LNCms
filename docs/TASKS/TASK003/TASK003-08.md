## 任务描述

实现系统配置的Redis缓存管理。包括：1) 实现配置读取优先从缓存获取的逻辑；2) 实现配置更新时清除缓存的逻辑；3) 实现缓存键的生成和管理。所有代码必须遵循Golang编码规范，错误消息使用中文。

## 依赖任务

- TASK003-05: 创建ConfigService接口和实现

## 实现指南

### 1. 定义缓存接口

定义缓存接口，用于抽象缓存操作。

### 2. 实现缓存键生成

实现缓存键的生成逻辑，使用配置类型作为键的一部分。

### 3. 修改ConfigService的GetConfig方法

修改GetConfig方法，优先从缓存获取配置，缓存未命中时从数据库获取并写入缓存。

### 4. 修改ConfigService的UpdateConfig方法

修改UpdateConfig方法，更新配置后清除对应的缓存。

### 5. 实现缓存过期时间管理

设置合理的缓存过期时间，使用常量定义。

## 验收标准

1. 配置读取优先从缓存获取
2. 配置更新时正确清除缓存
3. 缓存键生成逻辑正确
4. 代码规范符合要求
