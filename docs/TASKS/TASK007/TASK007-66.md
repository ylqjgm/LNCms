## 任务描述

扩展音频控制器，提供音频创建、更新、状态管理和删除的HTTP API处理。包括：1) 扩展AudioHandler接口；2) 实现CreateAudio、UpdateAudio、UpdateAudioStatus、DeleteAudio方法；3) 调用AudioService；4) 格式化响应（Envelope格式）。所有代码必须遵循API设计规范。

## 实现指南

### 1. 扩展AudioHandler接口

在 `internal/handler/audio_handler.go` 文件中扩展AudioHandler接口：

```go
// AudioHandler 音频控制器接口
// 提供音频管理的HTTP API处理
type AudioHandler interface {
    // GetAudios 获取音频列表（支持分页、筛选、排序）
    GetAudios(c *gin.Context)
    
    // GetAudio 根据ID获取音频
    GetAudio(c *gin.Context)
    
    // CreateAudio 创建音频（需要作者或管理员权限）
    CreateAudio(c *gin.Context)
    
    // UpdateAudio 更新音频（需要作者或管理员权限）
    UpdateAudio(c *gin.Context)
    
    // UpdateAudioStatus 更新音频状态（需要作者或管理员权限）
    UpdateAudioStatus(c *gin.Context)
    
    // DeleteAudio 删除音频（需要作者或管理员权限）
    DeleteAudio(c *gin.Context)
}
```

### 2. 定义请求结构体

在同一个文件中添加请求结构体：

```go
// CreateAudioRequest 创建音频请求
type CreateAudioRequest struct {
    Title        string   `json:"title" binding:"required"`
    Description  *string  `json:"description,omitempty"`
    AuthorID     int64    `json:"author_id" binding:"required"`
    CategoryIDs  []int64  `json:"category_ids,omitempty"`
    TagIDs       []int64  `json:"tag_ids,omitempty"`
    CoverURL     *string  `json:"cover_url,omitempty"`
    Status       string   `json:"status"`
    SerialStatus string   `json:"serial_status"`
    Rating       float64  `json:"rating"`
    IsActive     *bool    `json:"is_active"`
}

// UpdateAudioRequest 更新音频请求
type UpdateAudioRequest struct {
    Title        string   `json:"title" binding:"required"`
    Description  *string  `json:"description,omitempty"`
    AuthorID     int64    `json:"author_id" binding:"required"`
    CategoryIDs  []int64  `json:"category_ids,omitempty"`
    TagIDs       []int64  `json:"tag_ids,omitempty"`
    CoverURL     *string  `json:"cover_url,omitempty"`
    Status       string   `json:"status"`
    SerialStatus string   `json:"serial_status"`
    Rating       float64  `json:"rating"`
    IsActive     *bool    `json:"is_active"`
}

// UpdateAudioStatusRequest 更新音频状态请求
type UpdateAudioStatusRequest struct {
    Status string `json:"status" binding:"required"`
}
```

### 3. 实现CreateAudio、UpdateAudio、UpdateAudioStatus、DeleteAudio方法

参考NovelHandler和ComicHandler的实现方式，实现音频的创建、更新、状态管理和删除方法。实现逻辑与ComicHandler类似，但调用AudioService的方法。

### 5. 注意事项

- **API规范**：所有API必须遵循OpenAPI 3.1.1规范
- **响应格式**：所有响应必须使用Envelope格式
- **权限验证**：创建、更新、删除操作需要作者或管理员权限（参考NovelHandler的实现）
- **资源权限**：作者只能操作自己的音频，管理员可以操作所有音频
- **参数验证**：使用Gin的binding进行请求参数验证
- **错误处理**：所有错误消息必须使用中文
- **JWT认证**：从JWT token中获取用户ID和角色信息
