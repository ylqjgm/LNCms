### 2. 创建AudioSeason模型文件

在 `internal/models/` 目录下创建 `audio_season.go` 文件，定义AudioSeason模型：

```go
package models

import (
    "time"
    "gorm.io/gorm"
)

// AudioSeason 音频分季模型
// 存储音频作品的分季信息，用于组织和管理音频分集
// 一个音频作品可以包含多个分季，每个分季包含多个分集
type AudioSeason struct {
    ID          int64          `gorm:"primaryKey;column:id;type:BIGSERIAL" json:"id"`
    AudioID     int64          `gorm:"column:audio_id;type:BIGINT;not null;index:idx_audio_seasons_audio_id;index:idx_audio_seasons_audio_sort" json:"audio_id"`
    Name        string         `gorm:"column:name;type:VARCHAR(200);not null" json:"name"`
    Description *string        `gorm:"column:description;type:TEXT" json:"description,omitempty"`
    SortOrder   int            `gorm:"column:sort_order;type:INTEGER;not null;default:0;index:idx_audio_seasons_sort_order;index:idx_audio_seasons_audio_sort" json:"sort_order"`
    Status      string         `gorm:"column:status;type:VARCHAR(50);not null;default:'draft';index:idx_audio_seasons_status" json:"status"`
    CreatedAt   time.Time      `gorm:"column:created_at;type:TIMESTAMPTZ;default:CURRENT_TIMESTAMP" json:"created_at"`
    UpdatedAt   time.Time      `gorm:"column:updated_at;type:TIMESTAMPTZ;default:CURRENT_TIMESTAMP" json:"updated_at"`
    
    // 关联关系
    Audio     *Audio     `gorm:"foreignKey:AudioID;references:ID" json:"audio,omitempty"`
}

// TableName 指定表名
func (AudioSeason) TableName() string {
    return "audio_seasons"
}

// BeforeCreate 创建前钩子
func (as *AudioSeason) BeforeCreate(tx *gorm.DB) error {
    now := time.Now()
    if as.CreatedAt.IsZero() {
        as.CreatedAt = now
    }
    if as.UpdatedAt.IsZero() {
        as.UpdatedAt = now
    }
    return nil
}

// BeforeUpdate 更新前钩子
func (as *AudioSeason) BeforeUpdate(tx *gorm.DB) error {
    as.UpdatedAt = time.Now()
    return nil
}
```

### 3. 添加模型注释

在模型文件中添加详细的中文注释，说明每个字段的用途：

```go
// AudioSeason 音频分季模型
// 存储音频作品的分季信息，用于组织和管理音频分集
// 一个音频作品可以包含多个分季，每个分季包含多个分集
type AudioSeason struct {
    // ID 分季ID，自增主键
    ID int64 `gorm:"primaryKey;column:id;type:BIGSERIAL" json:"id"`
    
    // AudioID 音频ID，外键关联audios表，必填字段
    AudioID int64 `gorm:"column:audio_id;type:BIGINT;not null;index:idx_audio_seasons_audio_id;index:idx_audio_seasons_audio_sort" json:"audio_id"`
    
    // Name 分季名称，必填字段，最大长度200字符
    Name string `gorm:"column:name;type:VARCHAR(200);not null" json:"name"`
    
    // Description 分季描述，说明分季的内容和特点，可选字段
    Description *string `gorm:"column:description;type:TEXT" json:"description,omitempty"`
    
    // SortOrder 排序顺序，数字越小越靠前，用于排序显示，默认值为0
    SortOrder int `gorm:"column:sort_order;type:INTEGER;not null;default:0;index:idx_audio_seasons_sort_order;index:idx_audio_seasons_audio_sort" json:"sort_order"`
    
    // Status 分季状态：draft-草稿、published-已发布、archived-已归档
    // 必须符合检查约束，默认值为draft
    Status string `gorm:"column:status;type:VARCHAR(50);not null;default:'draft';index:idx_audio_seasons_status" json:"status"`
    
    // CreatedAt 创建时间，自动设置为当前时间
    CreatedAt time.Time `gorm:"column:created_at;type:TIMESTAMPTZ;default:CURRENT_TIMESTAMP" json:"created_at"`
    
    // UpdatedAt 更新时间，每次更新时自动更新
    UpdatedAt time.Time `gorm:"column:updated_at;type:TIMESTAMPTZ;default:CURRENT_TIMESTAMP" json:"updated_at"`
    
    // 关联关系
    // Audio 音频作品信息，外键关联audios表
    Audio *Audio `gorm:"foreignKey:AudioID;references:ID" json:"audio,omitempty"`
}
```

### 4. 定义模型方法

为AudioSeason模型添加常用的辅助方法：

```go
// IsPublished 检查分季是否已发布
func (as *AudioSeason) IsPublished() bool {
    return as.Status == "published"
}

// ValidateStatus 验证状态值是否有效
func (as *AudioSeason) ValidateStatus() bool {
    validStatuses := []string{"draft", "published", "archived"}
    for _, validStatus := range validStatuses {
        if as.Status == validStatus {
            return true
        }
    }
    return false
}
```

### 5. 注意事项

- **字段类型映射**：确保GORM标签中的类型与PostgreSQL数据库类型一致
- **索引定义**：在GORM标签中定义索引，包括audio_id、sort_order、status，以及复合索引 (audio_id, sort_order)
- **外键关联**：使用GORM的foreignKey和references标签定义与Audio的关联关系
- **JSON标签**：所有字段都应有JSON标签，可选字段使用omitempty
- **指针类型**：可选字段（如Description）使用指针类型，支持NULL值
- **时间字段**：使用time.Time类型，支持PostgreSQL的TIMESTAMPTZ类型
- **默认值**：在GORM标签中设置默认值，与数据库设计保持一致
- **检查约束**：status字段需要在应用层进行验证，确保符合数据库约束
- **排序字段**：sort_order用于控制分季的显示顺序
