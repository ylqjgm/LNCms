## 项目简介

LNCms 是一个现代化的内容发布网站系统，支持小说、漫画、音频、视频等多种内容类型。系统采用微服务架构设计，支持前后端分离、模块化部署，使用多种先进技术栈构建。

**注意**:
> 此项目属于试验性项目，为了测试 AI 的能力，以及自己对于 AI 的使用方法，而特意创建的项目。
> 本项目除部分文档外，所有内容均使用 AI 编写生成，不进行手动修改。
> 所有手动编写的文档，均会在文档中进行标注。

## 技术栈

- **前端语言**: JavaScript ES6

- **全文搜索**: Elasticsearch

## 项目说明

### 核心模块

- 核心模块 **Core** 为程序核心模块，负责提供整个项目业务逻辑处理、数据处理等核心功能。
- 核心模块执行后，提供 API 接口服务供前端模块进行数据交互。
- 核心模块执行后，通过 gRPC 协议与其余后端模块进行数据通信。

### 存储模块

- 存储模块 **Storage** 为程序远端存储模块，属于可选模块，未部署时，核心模块默认将文件按照配置方式存储于本地或云存储中。
- 存储模块部署后，通过 gRPC 协议与其他模块进行数据通信。
- 存储模块支持独立数据库，也支持与核心模块共用数据库。
- 存储模块提供文件压缩、转码服务，通过 Redis 队列进行排队处理。
- 存储模块支持数据迁移，支持分布式部署。

### 日志模块

- 日志模块 **Log** 为程序日志信息存储模块，属于可选模块，未部署时，其余模块仅支持默认的基本日志数据。
- 日志模块部署后，通过 gRPC 协议与其他模块进行数据通信。
- 日志模块支持独立数据库，也支持与核心模块共用数据库。
- 日志模块仅提供日志写入、查询功能。

### 采集模块

- 采集模块 **Crawler** 为程序远程采集模块，属于可选模块，未部署时，不支持数据采集功能。
- 采集模块部署后，通过 gRPC 协议与其他模块进行数据通信。
- 采集模块支持远程文件下载，支持直接下载文件到核心模块本地、存储模块、云存储。
- 采集模块不支持文件压缩、转码服务。
- 采集模块与核心模块共用数据库，不支持独立数据库。
- 采集模块提供强大、灵活的采集规则定制，支持采集流程定制。

### 前端模块

- 前端模块 **Web** 为程序前端展示模块，属于可选模块，未部署时，无法通过可视化界面查看项目数据。
- 前端模块部署后，通过 API 与核心模块进行数据通信。
- 前端模块分为前端、后台，可对核心模块中提供的所有 API 进行接口操作。
